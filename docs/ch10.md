# Chapter 10 : A Unified CSI-IMU Perception Architecture for Indoor Drone Navigation

> **Objective**
>
> The previous chapters demonstrated that CSI contains multiple complementary representations of the wireless environment. Rather than relying on a single representation, this chapter proposes a conceptual architecture that integrates geometric, motion, and learned CSI features with inertial sensing to enable robust indoor drone navigation.

---

# 10.1 Motivation

Throughout this notebook, we studied three fundamentally different approaches to CSI processing.

- **Widar** extracts motion through Doppler analysis.
- **SpotFi** extracts geometric information through AoA and ToF estimation.
- **DeepFi** learns latent representations directly from CSI.

Although each approach performs well for its intended task, each captures only one aspect of the wireless channel.

This motivates a unified framework capable of preserving complementary information instead of selecting a single representation.

---

# 10.2 Lessons Learned

The main observations from the literature review are summarized below.

| Method | Primary Representation | Main Strength | Limitation |
|----------|-----------------------|---------------|------------|
| Widar | Doppler / Velocity | Motion understanding | Limited geometric information |
| SpotFi | AoA + ToF | High localization accuracy | Sensitive to calibration |
| DeepFi | Learned Embedding | Automatic feature extraction | Limited interpretability |

No single approach captures the complete physical information available in CSI.

---

# 10.3 Design Principles

The proposed framework is guided by four principles.

1. Preserve multiple CSI representations instead of selecting one.

2. Combine physics-based models with learned representations.

3. Fuse complementary sensing modalities rather than relying solely on Wi-Fi.

4. Support continuous state estimation for autonomous navigation.

# 10.5 CSI Processing Branch

The CSI branch consists of three complementary feature extractors.

### Geometry Branch (SpotFi)

Extracts

- Angle of Arrival (AoA)
- Time of Flight (ToF)
- Multipath structure

This branch provides spatial constraints describing the surrounding environment.

---

### Motion Branch (Widar)

Extracts

- Doppler spectrum
- Velocity spectrum
- Micro-Doppler signatures

This branch captures dynamic changes caused by moving objects or platform motion.

---

### Representation Branch (DeepFi)

Learns compact latent representations directly from CSI.

These embeddings capture statistical channel characteristics that may not be explicitly represented by handcrafted physical models.

---

# 10.6 IMU Processing Branch

The inertial branch processes measurements from

- Accelerometer
- Gyroscope

through IMU preintegration.

The IMU provides

- short-term motion estimation,
- orientation tracking,
- high-rate state updates,

which complement the lower-rate CSI observations.

---

# 10.7 Cross-modal Fusion

The outputs of the CSI branches and the IMU encoder are fused using a cross-modal transformer.

The objective is to learn relationships between

- geometry,
- motion,
- learned CSI representations,
- inertial measurements.

The fused representation provides a unified estimate of the environment and platform state.

---

# 10.8 State Estimation

The fused representation can support multiple downstream tasks.

- Indoor localization
- Trajectory estimation
- Environment mapping
- Dynamic obstacle awareness
- Autonomous drone navigation

Rather than solving a single problem, the architecture aims to provide a general perception framework.

---

# 10.9 Expected Advantages

Compared to existing CSI-based systems, the proposed framework offers several potential advantages.

- Preserves complementary information instead of discarding it.
- Combines physical interpretability with representation learning.
- Improves robustness through sensor fusion.
- Supports continuous navigation rather than isolated localization.
